Уточнения к ТЗ:

1. Будем рассматривать приложение как прототип системы управления задачами, 
которые ставят пользователи (сотрудники компании) администратору сети или сисадмину.

2. Само приложение будем рассматривать, как одностраничное (SPA - simple page application).
Следовательно, используем 1 контроллер - Tasks с методами index(), add(), edit(), notfound().

3. В базе данных создадим 4 таблицы. Основная таблица - tasks для хранения общего списка задач.
Базовыми параметрами будем считать id задачи, id пользователя, поставившего эту задачу;
id почтового аккаунта, введенного пользователем для этой задачи; контент (текст задачи), 
дату постановки задачи, дату закрытия задачи, id текущего статуса задачи. 
Вспомогательные таблицы - users со списком пользователей, statuses c реестром статусов,
emails со списком используемых почтовых аккаунтов. 
Связь один-ко-многим. Схема данных прилагается.

4. В качестве СУБД задействуем MySQL, в качестве библиотеки провайдера - mysqli. В качестве
классов моделей используем классы-отображения: User, Status, Email и Task. Включим модели в 
иерархию классов, производных от базового класса DbProvider, инкапсулирующего логику доступа 
к базе данных.
